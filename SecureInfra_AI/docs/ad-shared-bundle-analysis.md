# AD Shared Bundle Analysis

`ad-shared` analyzes a folder that contains multiple Active Directory and Group
Policy report files generated by the defensive audit scripts.

Phase 1.5 discovers the report bundle, loads known JSON files safely on Windows,
normalizes `inactive-users.json`, and lists other detected reports for future
normalizer support.

## Expected Input Folder

The input should be a directory such as:

```text
reports/ad-shared/
  inactive-users.json
  password-never-expires.json
  service-accounts.json
  spn-exposure.json
  stale-computers.json
  privileged-groups.json
  privileged-identity-protection.json
  gpo-health.json
```

CSV and Markdown files can exist in the same folder, but Phase 1.5 bundle
normalization focuses on known JSON report files.

## Supported Files

The bundle detector recognizes:

- `inactive-users.json`
- `password-never-expires.json`
- `service-accounts.json`
- `spn-exposure.json`
- `stale-computers.json`
- `privileged-groups.json`
- `privileged-identity-protection.json`
- `gpo-health.json`

Missing files do not stop analysis. They are listed in the normalized report
metadata and executive summary.

## Currently Normalized Files

`inactive-users.json` is converted into detailed normalized findings using the
existing Active Directory inactive users normalizer.

## Detected But Not Yet Normalized

These files are loaded and summarized when present, but detailed finding
normalizers are not implemented yet:

- `password-never-expires.json`
- `service-accounts.json`
- `spn-exposure.json`
- `stale-computers.json`
- `privileged-groups.json`
- `privileged-identity-protection.json`
- `gpo-health.json`

The analyzer adds a note for each detected file whose detailed normalizer is not
implemented. It does not invent findings.

## Example Command

```powershell
python .\SecureInfra_AI\scripts\reporting\secureinfra_analyzer.py `
  --input .\reports\ad-shared `
  --type ad-shared `
  --output .\reports\output `
  --language en `
  --format markdown
```

The analyzer writes:

- `normalized-report.json`
- `executive-summary.md`
- `technical-findings.md`
- `remediation-plan.md`

## Safety Notes

- The analyzer is report-only and does not modify Active Directory.
- JSON files are loaded with Windows-safe BOM handling.
- Only implemented normalizers produce detailed findings.
- Privileged, SPN-bearing, built-in, service, and system-managed accounts are
  not safe for auto-remediation.
- Human owner review and approved change control are required before account
  changes.
