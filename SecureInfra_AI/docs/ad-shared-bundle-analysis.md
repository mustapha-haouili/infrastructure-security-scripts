# AD Shared Bundle Analysis

`ad-shared` analyzes a folder that contains multiple Active Directory and Group
Policy report files generated by the defensive audit scripts.

The analyzer discovers the report bundle, loads known JSON files safely on
Windows, normalizes implemented report types, and reports missing optional
files without stopping analysis.

## Expected Input Folder

The input should be a directory such as:

```text
reports/ad-shared/
  inactive-users.json
  password-never-expires.json
  service-accounts.json
  spn-exposure.json
  stale-computers.json
  privileged-groups.json
  privileged-identity-protection.json
  gpo-health.json
```

CSV and Markdown files can exist in the same folder, but bundle normalization
focuses on known JSON report files.

## Supported Files

The bundle detector recognizes:

- `inactive-users.json`
- `password-never-expires.json`
- `service-accounts.json`
- `spn-exposure.json`
- `stale-computers.json`
- `privileged-groups.json`
- `privileged-identity-protection.json`
- `gpo-health.json`

Missing files do not stop analysis. They are listed in the normalized report
metadata and executive summary.

## Currently Normalized Files

These files are converted into detailed normalized findings:

- `inactive-users.json`
- `password-never-expires.json`
- `service-accounts.json`
- `spn-exposure.json`
- `stale-computers.json`
- `privileged-groups.json`
- `privileged-identity-protection.json`
- `gpo-health.json`

## Known Files Without Detailed Normalizers

None in the current AD shared bundle set. The analyzer does not invent findings
for unknown file names or unsupported shapes.

## Example Command

```powershell
python .\SecureInfra_AI\scripts\reporting\secureinfra_analyzer.py `
  --input .\reports\ad-shared `
  --type ad-shared `
  --output .\reports\output `
  --language en `
  --format markdown
```

The analyzer writes:

- `normalized-report.json`
- `executive-summary.md`
- `technical-findings.md`
- `remediation-plan.md`

## Safety Notes

- The analyzer is report-only and does not modify Active Directory.
- JSON files are loaded with Windows-safe BOM handling.
- Known AD/GPO JSON files are normalized into detailed findings when present.
- Privileged, SPN-bearing, built-in, service, and system-managed accounts are
  not safe for auto-remediation.
- GPO findings require owner review, backup or export, staged testing, and
  approved change control before changes.
- Human owner review and approved change control are required before account
  changes.
