# v1.1.0 - Active Directory and Group Policy Security Audit Improvements

Release target date: 2026-06-07

## Summary

Version `1.1.0` moves the repository from a collection of scripts toward a more
structured infrastructure security assessment toolkit. The release emphasizes
Active Directory and Group Policy audit coverage, documentation, reporting
standards, sample outputs, and a safety-first assessment model.

## Highlights

- Added Active Directory audit scripts for inactive users, stale computers,
  service accounts, SPN exposure, password-never-expires accounts, privileged
  group changes, and privileged identity protection gaps.
- Added Group Policy health reporting for GPO inventory, links, stale policies,
  filtering, legacy indicators, and version mismatch evidence.
- Added methodology documentation for audit-first security assessment workflow.
- Added service model documentation for structured infrastructure health checks.
- Added fictional sample reports and sample outputs.
- Added JSON schemas for findings, report families, host audits, and remediation
  plans.
- Added an initial Markdown report generator for JSON audit results.

## Safety Model

- Audit-only collection should be the default starting point.
- Dry-run mode should be used before remediation.
- Changes should require explicit `--apply`, `-Apply`, or equivalent approval.
- Reports should be reviewed before implementation.
- Production environments should be tested carefully before changes are applied.

## Known Limitations

- The report generator is an initial Markdown implementation and does not yet
  generate HTML or PDF output.
- JSON output formats still vary by script family.
- Cloud identity checks such as Entra ID MFA and Conditional Access are not yet
  validated by the on-prem Active Directory scripts.
- Sample reports are fictional and should not be used as evidence of a real
  environment.

## Planned Next Steps

- Continue standardizing JSON output across script families.
- Add richer report templates and HTML output.
- Add historical comparison support.
- Expand Windows and Linux health check coverage.
- Keep customer-specific reporting, portal, pricing, and commercial templates in
  a private layer.
