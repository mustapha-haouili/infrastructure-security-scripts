# v1.2.0-beta.1 - SecureInfra AI Bundle Analysis and Windows Scope Analyzers

Release target date: 2026-06-20

## Summary

Version `1.2.0-beta.1` moves the repository from a structured infrastructure
security scripts toolkit toward a beta-ready defensive assessment platform.

This release introduces a stronger `SecureInfra_AI` layer for local,
deterministic analysis of infrastructure security evidence. It adds client
bundle analysis, multi-bundle fleet analysis, historical comparison, correlation
support, a local dashboard, and beta standalone Windows report analyzers.

The release remains audit-first, defensive, local-by-default, and focused on
authorized infrastructure security assessments.

## Highlights

* Added `SecureInfra_AI` client bundle analysis for full client collection
  folders and zip archives.
* Added multi-bundle / fleet analysis for reviewing evidence from multiple
  machines or client bundles.
* Added local SecureInfra Dashboard for reviewing normalized JSON reports,
  severity metrics, evidence details, related findings, historical comparison,
  and fleet coverage.
* Added historical comparison support for repeated assessments, including new,
  persistent, and resolved finding views.
* Added deterministic cross-source correlation groups for related findings.
* Added schema validation before writing normalized JSON and Markdown reports.
* Added client-side collection workflow with
  `Start-SecureInfraClientCollection.ps1`.
* Added Windows local administrator inventory and RDP exposure audit scripts.
* Added Windows server, workstation, and network exposure evidence collection.
* Added beta standalone Windows analyzer types:

  * `windows-host-audit`
  * `windows-server-audit`
  * `windows-workstation-audit`
  * `windows-network-exposure`

* Added fictional Windows sample inputs for the new analyzer types.

## New SecureInfra AI Analyzer Types

This release adds the following beta standalone analyzer types:

| Analyzer type               | Purpose                                                   |
| --------------------------- | --------------------------------------------------------- |
| `windows-host-audit`        | Normalize Windows host security audit JSON evidence       |
| `windows-server-audit`      | Normalize Windows server security inventory evidence      |
| `windows-workstation-audit` | Normalize Windows workstation security inventory evidence |
| `windows-network-exposure`  | Normalize local Windows network exposure evidence         |

Example usage:

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/windows-security-audit.json \
  --type windows-host-audit \
  --output reports/secureinfra-ai-host
```

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/windows-server-security.json \
  --type windows-server-audit \
  --output reports/secureinfra-ai-server
```

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/windows-workstation-security.json \
  --type windows-workstation-audit \
  --output reports/secureinfra-ai-workstation
```

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/windows-network-exposure.json \
  --type windows-network-exposure \
  --output reports/secureinfra-ai-network
```

## Client Bundle Workflow

This release introduces a safe client evidence collection workflow.

The Windows collection launcher runs supported audit and dry-run checks, writes a
structured result folder, creates a manifest and summary, and packages the result
as a zip bundle for local reviewer-side analysis.

Example:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\windows\Start-SecureInfraClientCollection.ps1
```

Analyze a returned client bundle:

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/secureinfra-client-collection-CLIENT-20260619-120000.zip \
  --type client-bundle \
  --output reports/secureinfra-ai-client
```

Analyze multiple returned client bundles:

```bash
python3 SecureInfra_AI/scripts/reporting/secureinfra_analyzer.py \
  --input reports/client-bundles \
  --type multi-bundle \
  --output reports/secureinfra-ai-fleet
```

## Dashboard

This release adds a local static dashboard under:

```text
SecureInfra_AI/dashboard/index.html
```

The dashboard can review normalized JSON reports locally and display:

* Severity metrics
* Finding filters
* Evidence details
* Related finding links
* Historical comparison
* Fleet collection coverage
* Machine-level filtering for multi-bundle reports

The dashboard is local/static and does not provide a multi-user customer portal.

## Safety Model

* Defensive and audit-first by design.
* Local analysis by default.
* No credential collection.
* No password spraying.
* No Kerberos ticket requests.
* No exploit code.
* No persistence or evasion logic.
* Remediation remains dry-run or approval-based where applicable.
* Client bundles and reports should be reviewed before sharing.
* Public examples and samples are fictional.

## Data Handling

Collection bundles and generated reports may contain infrastructure metadata.

Do not commit or upload real customer bundles, production reports, credentials,
tokens, private certificates, or environment-specific secrets to the public
repository.

Use fictional sample data only for public examples, tests, documentation, and
release demonstrations.

## Known Limitations

* This is a beta release.
* The standalone Windows analyzers currently normalize one JSON evidence report
  at a time.
* Some schemas are still evolving toward stricter long-term contracts.
* The local dashboard is intended for local review, not multi-user case
  management.
* The public repository does not include commercial/private features such as:

  * branded PDF reports
  * customer portal
  * tenant management
  * private AI-assisted narratives
  * customer-specific reporting workflows
  * pricing or service delivery templates
