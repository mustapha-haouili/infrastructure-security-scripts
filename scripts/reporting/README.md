# Reporting Scripts

Reporting scripts convert audit outputs into readable summaries for
administrators and stakeholders.

Full cross-platform reference: [../../docs/script-reference.md](../../docs/script-reference.md)

## `generate-markdown-report.py`

Purpose: convert one or more JSON audit result files into a Markdown assessment
report.

What problem it solves: individual audit scripts can produce separate JSON
files. This script gives administrators a first combined report view grouped by
severity.

Requirements:

- Python 3.
- JSON input files generated by this toolkit or shaped like finding records.
- Write access to the output directory.

Required permissions:

- No elevated system permissions are required.
- The operator only needs read access to input files and write access to the
  selected output path.

Parameters:

| Argument | Description |
|---|---|
| `json_files` | One or more JSON audit result files. |
| `--output FILE` | Markdown output path. Default: `reports/assessment-report-TIMESTAMP.md`. |
| `--title TEXT` | Custom report title. |
| `-h`, `--help` | Show built-in help. |

Usage examples:

```bash
python3 scripts/reporting/generate-markdown-report.py examples/sample-output/windows/windows-host-audit.example.json
python3 scripts/reporting/generate-markdown-report.py examples/sample-output/active-directory/ad-health-check.example.json examples/sample-output/group-policy/gpo-health.example.json --output reports/example-assessment.md
python3 scripts/reporting/generate-markdown-report.py reports/windows-audit.json --title "Example GmbH Infrastructure Security Review"
```

Example output:

- Markdown report under `reports/` unless `--output` is provided.
- Executive summary with input count and extracted finding count.
- Severity count table.
- Technical findings grouped by severity.

Safety notes:

- The script is report-only and does not change systems.
- Input files may contain sensitive operational evidence. Review generated
  Markdown before sharing it outside the administrator team.

Limitations:

- The script is an initial Markdown generator.
- HTML, PDF, risk scoring, and historical comparison are planned future
  improvements.
- Mixed legacy JSON shapes are normalized best-effort and may need refinement as
  schemas mature.

Recommended next steps:

- Use schema-aligned JSON outputs where possible.
- Review Critical and High findings first.
- Use generated reports as a starting point for validation and remediation
  planning, not as automatic approval to change systems.
